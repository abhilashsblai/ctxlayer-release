# SEO and Social Preview Notes

This file records the discoverability choices for the public CTX Layer release
repository.

## Primary Search Positioning

CTX Layer should be described as:

```text
CTX Layer: context engineering, memory, governance, MCP, and impact analysis for AI coding agents like Codex.
```

The repo should not rely on the product name alone. Search-friendly language
should connect the name to current developer searches around AI coding agents,
context engineering, agent memory, MCP, AI governance, audit trails, and impact
analysis.

## Social Preview

GitHub repository Open Graph images are controlled in the GitHub web UI, not by
README HTML tags. Upload this file under repository settings:

```text
docs/assets/ctxlayer-social-preview.png
```

Path in GitHub:

```text
Settings -> Social preview -> Edit -> Upload an image
```

The image is 1280 by 640 pixels and under 1 MB, which matches GitHub's
recommended size for repository social previews.

## GitHub Pages

The `docs/` folder contains a static landing page with explicit SEO,
Open Graph, Twitter card, canonical URL, robots, sitemap, and JSON-LD metadata.
If GitHub Pages is enabled from `master` and `/docs`, the shareable page will be:

```text
https://abhilashsblai.github.io/ctxlayer-release/
```

For the GitHub repository URL itself, GitHub still controls the final OG tags.
