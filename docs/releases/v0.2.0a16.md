# CTX Layer 0.2.0a16

Released July 12, 2026.

This release repairs migration and rollback behavior found while exercising a
real 589 MB legacy project database.

## Corrections

- Equal legacy/split column sets are copied by explicit column name even when
  physical column order differs.
- Source and target parity hashes use the same declared column projection.
- Backup manifests exclude transient SQLite WAL, SHM, and journal files.
- Older sidecar-affected rollback kits have a narrow integrity-checked recovery
  path.
- Candidate-created split roots are journaled before mutation and removed on
  rollback only when absent from the original backup.
- Database sidecars are cleared before a verified image is restored.
- Candidate preflight no longer initializes mutating doctor/service paths
  before backup.
- Failed rollback journals are retried deterministically.
- Migration SQLite connections are explicitly closed.

## Real Database Rehearsal

An isolated online copy of a 589 MB legacy project database completed:

- 142 authoritative tables copied;
- 530 rows copied with parity;
- 99 canonical records projected;
- split catalog activation;
- source-index rebuild;
- final project verification.

## Install Or Update

```powershell
python -m pip install --upgrade "https://github.com/abhilashsblai/ctxlayer-release/releases/download/v0.2.0a16/ctxlayer-0.2.0a16-py3-none-any.whl"
```

Installations already on `0.2.0a15` can use the normal path:

```text
ctxlayer update --check
ctxlayer update --dry-run
ctxlayer update
```
